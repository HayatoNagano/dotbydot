# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## プロジェクト概要

「dot by dot」はDead by Daylightにインスパイアされた2Dドット絵の非対称型対戦ホラーゲーム。ブラウザ上で動作する。1v1（サバイバー vs キラー）で、ローカル2P分割画面またはCPU対戦の一人プレイに対応。TypeScriptとCanvas 2D APIのみで構築し、ゲームエンジンやフレームワークは不使用。

## コマンド

- `npm run dev` — Vite開発サーバー起動
- `npm run build` — `tsc`で型チェック後、Viteでプロダクションビルド
- `npx tsc --noEmit` — 型チェックのみ
- `npx vitest run` — 全テスト実行
- `npx vitest run src/__tests__/FogOfWar.test.ts` — 単一テストファイル実行
- `npx vitest` — ウォッチモードでテスト実行

## アーキテクチャ

### ゲームループ
`main.ts`が`GameLoop`を生成（固定タイムステップ1/60秒 + 描画補間）。2つのフェーズ: **メニュー**（タイトル → モード選択 → キャラ選択）と**ゲーム**（対戦）。`Game`は毎試合再生成、`Input`は試合を跨いで共有。

### 更新サイクル (`Game.update`)
毎ティック: 入力読み取り（またはAI出力） → 能力 → 移動+タイル衝突 → インタラクション（修理/フック/板/ロッカー） → 投射物更新（斧、罠） → 霧再計算 → カメラ追従 → チェイス検出 → 勝敗判定。`Game`クラスが全エンティティとシステムを直接保持する中央オーケストレーター（ECSではない）。

### 座標系
- **タイル座標**: 整数グリッド（50×50）、`TileMap`・`FogOfWar`・`Pathfinding`で使用
- **ピクセル座標**: タイル × 48px（`TILE_SIZE = 48`）、エンティティの位置・移動・衝突で使用
- 変換: `tileX = Math.floor(px / TILE_SIZE)`

### レンダリング
`Renderer.renderView()`が1プレイヤー分のビューポートを描画: タイル → ワールドオブジェクト → 傷跡 → キャラクター → 霧エッジ → テラーレイディアスオーバーレイ。ローカル2Pではクリッピングで2回呼び出し（左半分=サバイバー、右半分=キラー）。CPU対戦ではフル幅で1回。

キャラクター描画は`fillRect`ベースのドット絵で、方向別（前/後/左/右）に異なるポーズを描画。サバイバーは健康状態ごとに外見が変化（健康→負傷→瀕死→死亡）。キラーはフード付きローブ、光る赤い目、武器を持つ。

### タイトル画面
- `PixelTitle.ts`: 7×9ビットマップフォントで「DOT BY DOT」を描画。血の滴り、フック/ナイフアイコン、3Dエフェクト付き
- `CampfireScene.ts`: 焚き火を囲む2人のサバイバーと、遠くの木陰から覗くキラーのシーン
- `Menu.ts`: 多層霧、脈動するビネット、地面の霧、スキャンラインオーバーレイ

### 霧（Fog of War）
プレイヤーごとの`boolean[]`グリッド。プレイヤーがタイル境界を跨いだ時のみ再計算（毎フレームではない）。半径内のBFSフラッドフィル + ブレゼンハム法による壁への視線チェック。端の壁は見えるが、その向こうは見えない。

### エンティティ階層
`Entity`（id, pos, size） → `Character`（移動、衝突、方向、補間） → `Survivor` / `Killer`。ワールドオブジェクト（`Generator`、`Hook`、`Pallet`、`Locker`、`ExitGate`、`Trap`、`Axe`）は`Entity`を直接継承。

### 板（Pallet）システム
部屋間のドアウェイ（2タイル開口部）に配置。倒す前は全員通過可能。サバイバーが倒すと2タイル分に拡張してドアウェイを封鎖。キラーのみブロック（`ExtraCollisionCheck`コールバックパターンで実装）、サバイバーは通過可能。キラーはインタラクトで板を破壊可能。倒す時にキラーが近くにいるとスタンを与える。

### 能力（Ability）
抽象基底クラス`Ability`（クールダウン/持続時間/発動ライフサイクル）。具象: `SprintBurst`、`DeadHard`（サバイバー）、`TrapAbility`、`ThrowAxe`（キラー）。`TrapAbility`と`ThrowAxe`は生成した`Trap`/`Axe`投射物エンティティの配列を保持。

### AI
FSMベース、霧を尊重（チートなし）。`KillerAI`: 巡回 → 調査（傷跡） → 追跡 → 攻撃 → 搬送 → フック。反応遅延（0.4〜1.0秒）、巡回/調査時は歩行、能力使用は低頻度。`SurvivorAI`: 修理 → 逃走 → 脱出。両者とも`Pathfinding`（A*）でナビゲーション。AI出力は`{ dx, dy, interact, ability, walk }`で、プレイヤー入力と同じ形式。

### 音声 (AudioManager)
`audio/AudioManager.ts`のシングルトン。Web Audio APIで全音声を手続き的に合成（音声ファイル不使用）。最初のユーザー操作（keydown/click）で`AudioContext`を初期化。

- **BGM**: メニュー音楽（5層: ドローン/パッド/コーラス/ピアノ/ノイズ）、アンビエント（ドローン+テンション）、チェイス音楽（4層: ベースパルス/ディソナントスタブ/ストリング/パーカッション）
- **焚き火**: タイトル画面でパチパチ音（フィルタノイズ + ランダムクラック/ポップ）
- **ハートビート**: テラーレイディアス連動、距離でテンポ変化（300ms〜1000ms）
- **SE**: 攻撃、ヒット、板（倒す/破壊）、ロッカー、発電機（修理ティック/完了）、スキルチェック（成功/大成功/失敗）、フック、罠、斧、ゲート通電、脱出、生贄、能力発動、メニュー操作
- **音楽遷移**: チェイス開始時にアンビエントをフェードダウン、チェイス終了時にフェードバック。メニュー↔ゲーム間でBGM切り替え

### HUD (InfoPanel)
`ui/InfoPanel.ts`のHTML DOM要素。画面下部に固定表示。サバイバー状態（体力/フック段階/行動）、目標（発電機/ゲート）、キラー状態（攻撃/スタン/搬送）、操作ガイドを表示。

### イベントシステム
`core/EventBus.ts`のシングルトン`EventBus`。`generator_completed`（閾値到達時にゲート通電を発火）と`survivor_sacrificed`で使用。

### キャラクター
- **サバイバー**: ドワット（スプリントバースト）、フェンリー（デッドハード）
- **キラー**: トラッパー（ベアトラップ）、ハントレス（斧投擲）

## 設計上の決定事項

- **外部依存ゼロ** — ランタイム依存なし、TypeScript/Vite/Vitestのみdev依存
- **定数は`constants.ts`に集約** — ゲームプレイのチューニング値を一元管理（TILE_SIZE=48, 移動速度等）
- **型・Enumは`types.ts`に集約** — 共有型定義
- **ドット絵はfillRect描画** — スプライトシートは不使用。キャラクターは方向別・状態別にピクセル単位で`fillRect`を組み合わせて描画（`p = Math.floor(S / 10)`をピクセル単位として使用）
- **シード付きRNG** — マップ生成・オブジェクト配置に決定的な乱数を使用し再現性を確保
- **Canvas clippingによる画面分割** — ビューポートごとに`ctx.save()` / `ctx.clip()` / `ctx.translate()`、別Canvasは使わない
- **ExtraCollisionCheckパターン** — `Character.move()`にオプショナルコールバックを渡すことで、タイルマップ以外の動的障害物（倒れた板）との衝突を実現。キラーのみに適用
- **全音声を手続き合成** — Web Audio APIのオシレーター、ノイズバッファ、フィルター、LFOで全サウンドを生成。音声ファイルは一切不使用
